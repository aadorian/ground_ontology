# Contributing to Grounded Research Ontology

Thank you for your interest in contributing! This page provides guidelines for contributing to the Grounded Research Ontology.

## ü§ù How to Contribute

### Reporting Issues

- **Bugs**: Use the [Bug Report template](https://github.com/aadorian/ground_ontology/issues/new?template=bug_report.yml)
- **Feature Requests**: Use the [Feature Request template](https://github.com/aadorian/ground_ontology/issues/new?template=feature_request.yml)
- **SWRL Rules**: Use the [SWRL Rules Request template](https://github.com/aadorian/ground_ontology/issues/new?template=swrl_rules_request.md)

### Contributing Code

1. **Fork the repository**
2. **Create a branch**: `git checkout -b feat/your-feature-name`
3. **Make your changes**
4. **Follow Conventional Commits**: See [CONTRIBUTING.md](../CONTRIBUTING.md)
5. **Submit a Pull Request**

## üìù Contribution Guidelines

### Commit Messages

Follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Types:**
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation
- `style`: Formatting
- `refactor`: Code refactoring
- `test`: Tests
- `chore`: Maintenance

**Examples:**
```
feat(ontology): add new class for research methodology
fix(parser): resolve null pointer exception
docs(readme): update installation instructions
```

### Pull Request Process

1. **Update documentation** if needed
2. **Test your changes** in Protege
3. **Ensure consistency** with existing ontology structure
4. **Follow the PR template** in your pull request

## üéØ Areas for Contribution

### Ontology Development

- **New Classes**: Add classes that fit the research methodology model
- **New Properties**: Add relationships between classes
- **Class Definitions**: Improve class definitions and constraints
- **SWRL Rules**: Add rules for automated reasoning

### Documentation

- **Wiki Pages**: Improve or add wiki pages
- **Examples**: Add more use cases and examples
- **Tutorials**: Create step-by-step tutorials
- **Translations**: Translate documentation to other languages

### Queries

- **SPARQL Queries**: Add new SPARQL queries
- **DL Queries**: Add new DL queries
- **Definition Queries**: Add new class definitions
- **Query Documentation**: Improve query documentation

### Tools and Automation

- **Scripts**: Add utility scripts
- **Validation**: Add validation tools
- **Documentation Generation**: Improve documentation automation

## üîç Code Review Process

1. **Automated Checks**: PRs are checked for consistency
2. **Maintainer Review**: At least one maintainer reviews
3. **Feedback**: Address any feedback
4. **Merge**: Once approved, PR is merged
